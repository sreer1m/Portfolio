🔍 Sources Tab: Debugging JavaScript

What is the Sources Tab?

The Sources tab is your debugging headquarters. It's where you can:

  • See all files that make up the website (HTML, CSS, JavaScript, images)
  • Pause JavaScript code while it's running
  • Step through code line by line
  • Watch variables change in real-time
  • Find bugs and understand how code works

💡 Think of it like this:

If you're watching a movie and want to understand a complex scene, you can pause, rewind, and watch frame by frame. The Sources tab lets you do the same thing with JavaScript code!

Sources Tab Layout

The Sources tab has three main sections:

📁

File Navigator

Left panel - shows all website files

📝

Code Editor

Middle panel - displays and edits code

🐛

Debugger Panel

Right panel - debugging controls and info

Breakpoints - Pausing Code Execution

A breakpoint is like a pause button for your code. When JavaScript reaches a breakpoint, it stops and waits for you to inspect what's happening.

How to Set a Breakpoint:

  1. Open Sources tab
  2. Find a JavaScript file in the file navigator
  3. Click on a line number in the code editor
  4. A blue marker appears - that's your breakpoint!

Stepping Through Code

Once code is paused at a breakpoint, you can control execution:

▶️

Resume (F8)

Continue running until next breakpoint

⤵️

Step Over (F10)

Run current line and move to next

⬇️

Step Into (F11)

Go inside function calls

⬆️

Step Out (Shift+F11)

Finish current function

Watch Expressions

Watch specific variables or expressions as code runs:

  1. While paused at a breakpoint, look at the right panel
  2. Find the "Watch" section
  3. Click the + button
  4. Type a variable name or expression
  5. It will update as you step through code!

💡 What to Watch

Good things to watch: variable values, object properties, array lengths, calculation results. Example: if you have a variable total, add it to Watch to see it change!

Call Stack

The Call Stack shows you how you got to the current line of code - which functions called which.

Think of it like a trail of breadcrumbs showing the path through your code.

function first() {
    second();
}

function second() {
    third();  // Breakpoint here
}

function third() {
    // When paused here, call stack shows:
    // third() ← second() ← first()
}

Scope Variables

While debugging, the Scope panel shows all available variables:

  • Local: Variables in the current function
  • Closure: Variables from parent functions
  • Global: Variables accessible everywhere (like window)

Key Takeaways

  • ✅ Sources tab shows all website files
  • ✅ Breakpoints pause code execution
  • ✅ Step controls let you walk through code line by line
  • ✅ Watch expressions to track specific values
  • ✅ Call Stack shows the path through your code
  • ✅ Scope shows all available variables at the current point
  • ✅ This is the #1 tool for understanding and fixing JavaScript!
