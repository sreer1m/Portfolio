🌐 Network Tab: Watching Website Traffic

What is the Network Tab?

The Network tab shows you EVERYTHING your browser downloads to display a web page. Every image, CSS file, JavaScript file, font, and data request - you can see it all!

💡 Think of it like this:

When you visit a website, your browser is like a customer at a restaurant. The Network tab is a receipt showing every single item ordered: "1 HTML page, 3 images, 2 CSS files, 5 JavaScript files..." It shows what was requested, how long it took, and how big each item was.

Opening the Network Tab

  1. Open DevTools (F12)
  2. Click the "Network" tab
  3. Important: Refresh the page (F5 or Ctrl+R) to see requests!
  4. You'll see a waterfall of all resources loading

⚠️ Important Note

The Network tab only records activity while it's open. If you want to see what loads on a page, open Network FIRST, then refresh the page!

Understanding Network Requests

Each line in the Network tab is a request - your browser asking a server for something.

Key Columns:

  • Name: What file was requested
  • Status: Success code (200 = OK, 404 = Not Found)
  • Type: document, stylesheet, script, image, etc.
  • Initiator: What triggered this request
  • Size: How big the file is
  • Time: How long it took to download
  • Waterfall: Visual timeline of the request

HTTP Status Codes

Status codes tell you if a request was successful or not:

2️⃣0️⃣0️⃣

200 - OK

Success! Everything worked

3️⃣0️⃣4️⃣

304 - Not Modified

Using cached version (faster!)

4️⃣0️⃣4️⃣

404 - Not Found

The file doesn't exist

5️⃣0️⃣0️⃣

500 - Server Error

Server had a problem

Request Details

Click on any request in the Network tab to see detailed information:

Headers Tab

  • Request Headers: Information your browser sends (browser type, cookies, etc.)
  • Response Headers: Information the server sends back (content type, cache rules, etc.)

Response Tab

Shows the actual content that was received - HTML, JSON, image data, etc.

Timing Tab

Shows exactly how long each phase of the request took

Throttling - Simulating Slow Connections

The Network tab can simulate slow internet connections to test how your site performs!

  1. In the Network tab, look for the throttling dropdown (usually says "No throttling")
  2. Choose options like "Fast 3G", "Slow 3G", or "Offline"
  3. Refresh the page to see how it loads on a slow connection
  4. This helps you understand the experience for users with slow internet

💡 Why This Matters

Not everyone has fast internet! Testing with throttling helps you make websites that work well for everyone, even on slower connections.

Download Size vs Transfer Size

You might see two sizes for each request:

  • Size: The actual file size
  • Transferred: How much was actually downloaded

If transferred is smaller or says "cached", it means the browser already had that file and didn't need to download it again!

Finding Performance Issues

The Network tab helps identify problems:

  • 🐌 Files taking too long to load
  • 📦 Files that are too large
  • ❌ Failed requests (404, 500 errors)
  • 🔄 Too many requests (lots of small files)
  • ⏳ Requests blocking other requests

Key Takeaways

  • ✅ Network tab shows all browser requests
  • ✅ Open it BEFORE loading the page to capture requests
  • ✅ Status codes tell you if requests succeeded (200 = OK, 404 = Not Found)
  • ✅ Click on requests to see headers, response data, and timing
  • ✅ Filter by type to focus on specific resources
  • ✅ Use throttling to test slow connections
  • ✅ Great for understanding APIs and debugging loading issues
